# ニュースキュレーション

最新のテックニュースを複数ソースから取得し、ユーザーの好みに基づいてキュレーションする。

**重要: 各ステップの開始時に進捗をユーザーに表示すること。** 処理が長いため、今何をしているかを逐次伝える。

## 手順

### 1. 好みファイルの読み込み

`.claude/skills/curate-news/preferences.md` が存在すれば読み込む。存在しなければデフォルトの好み（テック全般、全ソース有効）で進める。

好みファイルには以下が含まれる（存在する場合）:
- **興味・関心**: キュレーション時の優先度付けとカテゴリ自動選択に使う
- **品質基準**: キュレーション時のフィルタリングに使う

→ ユーザーに表示: `📋 好みファイルを読み込みました`

### 2. ソース定義の読み込み

→ ユーザーに表示: `📂 ソース定義を読み込んでいます...`

1. `.claude/skills/curate-news/sources/*.md` を Glob で検索し、全ファイルを Read で読み込む
2. 各ソースにはカテゴリごとのフィードURLと「含まれるトピック」が定義されている
3. **好みファイルの「興味・関心」と各カテゴリの「含まれるトピック」をマッチングし、関連度の高いソース・カテゴリを自動選択する**
4. 好みファイルがない場合は全ソースから幅広く取得する（各ソースの汎用的なカテゴリ1つずつ）
5. 1つのソースから複数カテゴリが選ばれてもよい（興味が複数のカテゴリにまたがる場合）
6. どのソース・カテゴリも興味と関連しなければ、そのソースは取得をスキップする

→ ユーザーに表示: `✅ {N}ソース / {M}フィードを取得対象にしました`（マッチング結果の表も表示）

### 3. キャッシュの確認とフィード取得

キャッシュディレクトリ: `.claude/skills/curate-news/cache/`
キャッシュのキー: `{ソースID}-{カテゴリ}.json`
**注意: カテゴリ名に `/` が含まれる場合はファイル名では `-` に置換する**

**まず Bash で現在時刻を取得する（モデルは現在時刻を知らないため必須）:**
```bash
date +%s
```

→ ユーザーに表示: `🔍 キャッシュを確認中...`

1. 全フィードのキャッシュファイルを Read で確認する
2. `fetched_at` を UNIX タイムスタンプに変換し、`fetched_at + ttl_minutes * 60` が現在時刻より未来ならキャッシュ有効
   ```bash
   date -j -f "%Y-%m-%dT%H:%M:%S" "{fetched_at}" +%s
   ```
3. キャッシュ有効なフィード → ユーザーに表示: `✅ [{ソース名}] {カテゴリ} キャッシュを使用（{N}分前に取得）`
4. 期限切れ or ファイルなしのフィード → 再取得が必要

**再取得が必要なフィードは Agent ツールで haiku の subagent に委譲する。**
各 subagent にソース定義の内容とキャッシュパスを渡し、取得→パース→キャッシュ保存まで任せる。
**複数フィードは複数の subagent を並列起動して同時取得する。**

→ ユーザーに表示: `📡 {N}フィードを並列取得中...`

各ソースの取得方法・パース仕様は `.claude/skills/curate-news/sources/*.md` に定義されている。**その仕様に従って取得・パースすること。** ソースごとにフィード形式（RSS、JSON API等）やフィールド名が異なるため、必ずソース定義を参照する。

#### subagent への指示テンプレート

```
以下のフィードを取得し、キャッシュに保存してください。

ソース定義:
{ソース定義の内容}

取得対象: {カテゴリ名} ({フィードURL})
キャッシュパス: .claude/skills/curate-news/cache/{ソースID}-{カテゴリ}.json

取得→パース→キャッシュJSON書き出しを1つの Bash コマンドで完結させてください。
タイムスタンプや一時ファイルなど全てpython3内で処理してください。
完了したらキャッシュファイルの件数を報告してください。
```

#### 取得の原則

- **subagent は `model: haiku` で起動する**（取得・パースは機械的処理のため）
- タイムスタンプや一時ファイルなど全てpython3内で処理する（シェル変数をpython3に渡す方式は展開失敗しやすいため避ける）
- ソースごとのフィード形式・フィールドマッピングは各ソース定義を参照して実装する

#### キャッシュファイル形式

- `source_id`, `category`, `feed_url`: ソース定義から
- `fetched_at`: ISO 8601 JST（python3内で生成）
- `ttl_minutes`: ソース定義の TTL を分に変換した値
- `items`: 配列。各アイテムは `title`, `url`, `description`, `published_at`, `meta` を持つ
- `meta` の内容はソースごとに異なる（ソース定義のフィールドマッピングを参照）

**取得したフィードは必ずキャッシュに保存する。スキップしない。**

→ ユーザーに表示: `💾 キャッシュを保存しました`
→ ユーザーに表示: `✅ 全フィードの取得が完了しました（合計 {N} 記事）`

### 4. キュレーション

→ ユーザーに表示: `🤖 {N}件の記事からキュレーション中...`

全ソースから集めた記事を統合し、以下の基準でキュレーションする:

- 好みファイルがあればその内容に基づいてフィルタ・優先度付け
- なければテック全般として、多様なトピックをバランスよく選ぶ
- **重複排除**: 同じURLの記事は1つにまとめる
- **10件程度** にピックアップする
- ソースの `meta` にスコア情報（ブックマーク数、投票数等）があれば判断材料にする
- **好みファイルの「品質基準」に従ってフィルタリングする**
- 興味・関心との関連度とスコアを総合的に判断し、順位付けする

### 5. 出力

**好みファイルの興味・関心に基づき、関連度が高い順にフラットに並べて出力する。海外ニュースも含めて全て日本語で説明する。**

```
## テックニュース ({取得日時} JST)
{ソース名の一覧}

- {タイトル} ({ソース名})
  {一言コメント}
  {URL}

- {タイトル} ({ソース名})
  {一言コメント}
  {URL}

- ...
```

**ルール:**
- 興味・関心との関連度が高い順に並べる（スコアも加味）
- 好みファイルの「品質基準」に従ってフィルタリングする
- 海外記事のタイトルは日本語に翻訳する
- 一言コメントは1行で簡潔に
- スコアはキュレーション時の内部判断に使い、出力には含めない
- ソース名はソース定義の「表示名」を使う

### 6. 結果の保存

出力した内容を `.claude/skills/curate-news/output/{YYYY-MM-DD}.md` に Write で保存する。
同じ日に複数回実行した場合は上書きする。
