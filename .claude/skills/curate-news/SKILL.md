# ニュースキュレーション

最新のニュースを複数ソースから取得し、ユーザーの好みに基づいてキュレーションする。

処理が長いため、各ステップの開始時に進捗をユーザーに表示する。表示例:

| タイミング | 表示 |
|---|---|
| 好み読み込み後 | `📋 好みファイルを読み込みました` |
| ソース選択後 | `✅ {N}ソース / {M}フィードを取得対象にしました`（マッチング結果の表も表示） |
| キャッシュ確認中 | `🔍 キャッシュを確認中...` |
| キャッシュ有効時 | `✅ [{ソース名}] {カテゴリ} キャッシュを使用（{N}分前に取得）` |
| フィード取得中 | `📡 {N}フィードを並列取得中...` |
| 取得完了後 | `✅ 全フィードの取得が完了しました（合計 {N} 記事）` |
| キュレーション中 | `🤖 {N}件の記事からキュレーション中...` |

## 手順

### 1. 好みファイルの読み込み

`.claude/skills/curate-news/preferences.md` が存在すれば読み込む。存在しなければデフォルトの好み（全ソース有効）で進める。

好みファイルには以下が含まれる（存在する場合）:
- 興味・関心: キュレーション時の優先度付けとカテゴリ自動選択に使う
- 品質基準: キュレーション時のフィルタリングに使う

### 2. ソース定義の読み込み

1. `.claude/skills/curate-news/sources/*.md` を Glob で検索し、全ファイルを Read で読み込む
2. 各ソースにはカテゴリごとのフィードURLと「含まれるトピック」が定義されている
3. 好みファイルの「興味・関心」と各カテゴリの「含まれるトピック」をマッチングし、関連度の高いソース・カテゴリを自動選択する
4. 好みファイルがない場合は全ソースから幅広く取得する（各ソースの汎用的なカテゴリを1つずつ）
5. 1つのソースから複数カテゴリが選ばれてもよい（興味が複数のカテゴリにまたがる場合）
6. どのソース・カテゴリも興味と関連しなければ、そのソースは取得をスキップする

### 3. キャッシュの確認とフィード取得

#### キャッシュの確認

キャッシュディレクトリ: `.claude/skills/curate-news/cache/`
キャッシュのキー: `{ソースID}-{カテゴリ}.json`（カテゴリ名の `/` は `-` に置換する。ファイルパスの区切りと衝突するため）

まず Bash で現在時刻を取得する（モデルは現在時刻を知らないため）:
```bash
date +%s
```

1. 全フィードのキャッシュファイルを Read で確認する
2. `fetched_at` を UNIX タイムスタンプに変換し、`fetched_at + ttl_minutes * 60` が現在時刻より未来ならキャッシュ有効
   ```bash
   date -j -f "%Y-%m-%dT%H:%M:%S" "{fetched_at}" +%s
   ```
3. 期限切れまたはファイルなしのフィードを再取得する

#### フィード取得

再取得が必要なフィードは Agent ツールで haiku の subagent に委譲する（取得・パースは機械的処理のため）。複数フィードは複数の subagent を並列起動する。

各ソースの取得方法・パース仕様は `.claude/skills/curate-news/sources/*.md` に定義されている。ソースごとにフィード形式（RSS、JSON API等）やフィールド名が異なるため、ソース定義に従って取得・パースする。

subagent への指示テンプレート:

```
以下のフィードを取得し、キャッシュに保存してください。

ソース定義:
{ソース定義の内容}

取得対象: {カテゴリ名} ({フィードURL})
キャッシュパス: .claude/skills/curate-news/cache/{ソースID}-{カテゴリ}.json

取得→パース→キャッシュJSON書き出しを1つの Bash コマンドで完結させてください。
タイムスタンプや一時ファイルなど全てpython3内で処理してください。
完了したらキャッシュファイルの件数を報告してください。
```

取得の原則:
- subagent は `model: haiku`、`mode: bypassPermissions` で起動する（メインエージェントと同じ権限で動作させる）
- 同時起動数が多すぎると `bypassPermissions` でも権限エラーになる場合がある。その場合はソースをまとめて1つの subagent に複数フィードを処理させるか、バッチ数を減らして対応する
- タイムスタンプや一時ファイルなど全てpython3内で処理する。シェル変数をpython3に渡す方式は展開失敗しやすいため避ける

#### キャッシュファイル形式

取得したフィードは全てキャッシュに保存する。

- `source_id`, `category`, `feed_url`: ソース定義から
- `fetched_at`: ISO 8601 JST（python3内で生成）
- `ttl_minutes`: ソース定義の TTL を分に変換した値
- `items`: 配列。各アイテムは `title`, `url`, `description`, `published_at`, `meta` を持つ
- `meta` の内容はソースごとに異なる（ソース定義のフィールドマッピングを参照）

### 4. キュレーション

全ソースから集めた記事を統合し、以下の基準でキュレーションする:

- 好みファイルがあればその内容に基づいてフィルタ・優先度付け
- 同じURLの記事は1つにまとめる（重複排除）
- 10件程度にピックアップする（内訳は下記）
- ソースの `meta` にスコア情報（ブックマーク数、投票数等）があれば判断材料にする
- 好みファイルの「品質基準」に従ってフィルタリングする
- 興味・関心との関連度とスコアを総合的に判断し、順位付けする

#### 通常枠（8件程度）とワイルドカード枠（2件程度）

10件のうち8件は通常枠（興味・関心 × スコアで選ぶ）、2件はワイルドカード枠とする。

ワイルドカード枠の選定基準:
- ユーザーの興味・関心に直接マッチしなくてもよい
- 「普段読まないけど面白そう」「意外な切り口」「新しい分野の芽」を優先
- スコアの高さは不問（むしろ低スコアでもユニークなら歓迎）
- 宣伝記事は除外（品質基準は適用する）
- 通常枠と異なるソースやカテゴリから選ぶと多様性が増す

### 5. 出力

海外ニュースも含めて全て日本語で出力する。

```
## ニュース ({取得日時} JST)

{今日の概況 — 3〜4文}

1. [{タイトル}]({URL}) ({ソース名})

2. [{タイトル}]({URL}) ({ソース名})

...
```

今日の概況の書き方:
- 取得した全記事（選ばなかったものも含む）を俯瞰して、今日の空気感を伝える
- 例: どんなトピックが多いか、何がホットか、目立つトレンドや共通テーマがあるか
- その上で、今回はどういう観点でピックアップしたかを軽く添える
- 簡潔に。3〜4文程度

ルール:
- 通常枠とワイルドカード枠を区別せず、フラットに混ぜて並べる（関連度順にこだわらず、読んで面白い流れになるよう配置する）
- 海外記事のタイトルは日本語に翻訳する
- スコアはキュレーション時の内部判断に使い、出力には含めない
- ソース名はソース定義の「表示名」を使う

### 6. 結果の保存

出力した内容を `.claude/skills/curate-news/output/{YYYY-MM-DD}.md` に Write で保存する。
同じ日に複数回実行した場合は上書きする。
